Agent Directive: Force Strapi API Schema Sync & Fix "Invalid Key project"
The Situation: The database column project_id now exists and data is linked, but the Strapi API still returns "Invalid key project." This means the Strapi Shadow Schema is out of sync with the physical database.

Task 1: The "Nuke & Rebuild" (Strapi Reset)

Delete the .cache and build folders in your Strapi directory.

Run npm run build followed by npm run develop.

Why: Strapi generates a snapshot of your API during the build process. If the relation was added later, the API might be looking at an old snapshot that doesn't include the "project" key.

Task 2: Content-Type Permissions Refresh

In the Strapi Admin, go to Settings > Roles > Public.

Uncheck all permissions for "Property" and "Project," then click Save.

Re-check find and findOne for both, then click Save again.

Why: This forces Strapi to re-calculate the available fields for the Public API.

Task 3: Revert Frontend to Native Relationship Filtering

Once the API stops returning "Invalid key," remove the projectToAreaMap fallback.

Update the query to: ?filters[project][name][$eq]=Downtown Views&populate=*.

Verify that the response JSON now contains a project object inside the attributes (or at the root if using Strapi v5).

Task 4: Check schema.json in the API folder

Open src/api/property/content-types/property/schema.json.

Confirm the project relation is defined there. If it is only in the dist/ folder but not the src/ folder, your changes will be overwritten every time the server restarts.
